# Notifications Log

**Menu Path:** `Communication > Notifications`

The **Notifications** module acts as an audit trail for all systemic alerts sent to your users. It displays a chronological log of Push Notifications (if you have a mobile app) and In-App browser alerts.

## Using the Notifications Log

You generally do not "create" notifications here; they are triggered by the system (e.g., "Your order has shipped", "Your ticket was answered"). This module is used for monitoring and debugging.

### 1. Delivery Audits
If a customer claims they never received an update about their order:
- Search for the user's name or email in the Notifications list.
- Check the log to see exactly when the system dispatched the alert.
- Verify the **Status** column to see if the push notification was successfully delivered to their device or if it failed.

### 2. Broadcasts (Manual Push)
Depending on your platform's configuration, administrators may have a **"Send Notification"** button here.
- This allows you to write a custom, platform-wide message (e.g., "System maintenance tonight at 2 AM").
- You can target this broadcast to specific user segments (All Users, Merchants Only, VIPs).
