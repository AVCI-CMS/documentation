# Shipment & Fulfillment

**Menu Path:** `Finance & Logistics > Shipment`

The **Shipment** module bridges the gap between digital orders and physical logistics. It allows warehouse staff to manage dispatching and track packages without needing to look at financial data.

## The Fulfillment Workflow

### 1. Generating Shipping Labels
When orders are in the "Processing" state, the warehouse team uses this module:
1. Select the orders that are packed and ready to go.
2. Click **"Create Shipment"**.
3. If your CMS is integrated with a shipping provider (e.g., FedEx, UPS, Yurtiçi Kargo), this action sends an API request to the courier.
4. The courier returns a Tracking Number and a printable Barcode Label.
5. Click **"Print Labels"** to bulk-print the stickers to place on the boxes.

### 2. Tracking Deliveries
The Shipment list provides a specialized view of logistics statuses:
- **Label Created:** Box is ready, waiting for courier pickup.
- **In Transit:** The courier has scanned the box.
- **Out for Delivery:** The package is on the local delivery truck.
- **Delivered:** Handed to the customer.

> **Automated Order Updates:** When a shipment status changes to "Delivered" via the courier's API, AVCI CMS will automatically update the parent Order status to "Delivered" as well, closing the loop.

## Handling Returns (RMA)
If a customer initiates a return, a Return Shipment record is created here.
- You can generate a "Return Label" for the customer to print.
- Track the package as it travels *back* to your warehouse.
- Once marked as "Received" by your warehouse staff, the finance team is notified to process the refund.
