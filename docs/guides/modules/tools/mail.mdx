# Mail

**Menu Path:** `Tools > Mail`

While the *Newsletter* module is for marketing broadcasts, the **Mail** module is the technical backbone for all transactional emails generated by the CMS (e.g., "Order Confirmations", "Password Resets", "Welcome Emails").

## Key Features

### 1. SMTP Configuration
Before the CMS can send any emails, you must configure the outgoing mail server.
- Enter your **SMTP Host**, **Port**, **Username**, and **Password**.
- Set the **"From Name"** and **"From Email"** (e.g., `noreply@yourdomain.com`).
- *Tip:* Always use the "Send Test Email" button to verify your configuration is correct before going live.

### 2. Email Templates
The system comes with predefined templates for system events.
- You can edit the HTML of these templates to match your brand's colors and add your corporate logo.
- Use dynamic tags like `{{FIRST_NAME}}` or `{{ORDER_NUMBER}}` which the system will automatically replace with real data when sending the email.

### 3. Mail Logs
The module maintains a history of all emails sent by the system.
- If a customer claims they didn't receive their invoice, you can check the **Logs** to verify if the email was successfully delivered, bounced, or marked as spam by the receiving server.
