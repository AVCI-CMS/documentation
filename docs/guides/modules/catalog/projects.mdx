# Projects

**Menu Path:** `Catalog > Projects`

The **Projects** module is a portfolio management tool designed to showcase large-scale deliverables. Unlike products, projects are typically not "sold" through a shopping cart but are displayed to build credibility, attract leads, or show case studies (e.g., real estate developments, software implementations, architectural models).

## Creating a Project

To add a new project to your portfolio:

1. Navigate to **Catalog > Projects**.
2. Click the **"Add New"** button.
3. Fill out the project details across the following sections:

### 1. General Info & Links
- **Title:** The name of the project (e.g., "Skyline Residential Complex").
- **Slug:** URL identifier.
- **Status:** *Draft* (still editing), *Published* (visible on the site).
- **Category:** Categorize the project (e.g., "Completed Developments", "Concept Designs").
- **External Page Link:** If the project has its own dedicated website or external landing page, provide the URL here.
- **Social Media Links:** Link to external galleries or project-specific social media profiles.

### 2. Media
- **Cover Image:** The main hero image that represents the project on the portfolio grid.
- **Gallery:** High-quality images showcasing different angles, phases, or details of the project.

### 3. Specifications
- Technical details or project metrics. (e.g., "Total Area: 10,000 sqm", "Duration: 18 Months", "Client: ABC Corp"). These attributes are managed via the [Common Attributes](../common/attributes) module.

### 4. Description (Rich Text)
- Provide a comprehensive case study. Explain the problem, the solution, the technologies/materials used, and the final outcome. Use the rich text editor to format headers and embed videos.

4. Click **"Save"** to publish the project to your frontend portfolio.
