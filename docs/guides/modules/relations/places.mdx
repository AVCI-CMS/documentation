# Places

**Menu Path:** `Relations > Places`

The **Places** module acts as a global directory for physical locations. Instead of typing the same address details every time you create an Event or a Route, you create the "Place" once here and link it. This ensures absolute consistency across the platform.

## Creating a Place

1. Navigate to **Relations > Places**.
2. Click **"Add New"**.

### 1. Basic Info
- **Name:** The name of the venue or store (e.g., "Grand Stadium", "Downtown Branch").
- **Slug:** Auto-generated URL identifier for the place.
- **Status:** Set to *PUBLISHED* to make it available for linking.

### 2. Location & Map (Critical)
- **Address:** Enter the full physical address.
- **Coordinates (Latitude & Longitude):** You can enter these manually, or use the integrated Google Maps (or similar map provider) widget. Dragging the pin on the map will automatically update the coordinates.
- *Why is this important?* Modules like [Routes](../catalog/routes) rely heavily on these exact coordinates to calculate distances, transit times, and generate map previews.

### 3. Media & Details
- **Images:** Upload photos of the venue (exterior, interior).
- **Capacity:** The maximum number of people the place can hold. This acts as an upper limit when selling tickets for an Event at this location.
- **Description:** Facilities available (e.g., "Wheelchair accessible, Parking available").

3. Click **"Save"**. You can now select this venue from a dropdown when creating Events or Routes.
