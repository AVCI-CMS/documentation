# Places (Venues & Locations)

**Menu Path:** `Relations > Places`

The **Places** module serves as a geographical directory for your platform. It is used to define and manage physical locations such as event venues, physical store branches, pickup points, or warehouses.

## Why Define Places?
Instead of manually typing an address every time you create a physical Event, you define the venue once in this module. This ensures mapping coordinates are exact and reusable.

## Creating a New Place

1. Navigate to the Places list and click **"Create"**.
2. **Name:** Enter the recognizable name of the location (e.g., "Grand Convention Center" or "Taksim Branch").
3. **Full Address:** Type the complete street address. This text is usually displayed to the end-user.
4. **Map Coordinates:** 
   - Enter the exact Latitude and Longitude. 
   - Many CMS integrations allow you to drop a pin on an embedded Google Map to auto-fill these coordinates. This is vital for generating the "Get Directions" links on the frontend.
5. **Venue Attributes:** (Optional) Define the maximum seating capacity, parking availability, or accessibility features.

Once saved, this "Place" becomes a selectable option in [Create an Event](../catalog/events/create.mdx) module and Route planner.
