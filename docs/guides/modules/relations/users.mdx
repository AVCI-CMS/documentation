# Users

**Menu Path:** `Relations > Users`

The **Users** module is your primary directory for all registered end-customers (B2C) or buyers. It provides a comprehensive 360-degree view of a user's interaction with your platform.

## Viewing a User Profile

1. Go to **Relations > Users**.
2. Click on a User ID or name to open their profile.

### 1. Basic Information
- **Personal Details:** First Name, Last Name, Email, and Phone Number.
- **Status:** You can manually block a user by changing their status from *ACTIVE* to *SUSPENDED* if they violate terms of service.
- **Registration Date:** When the user first signed up.

### 2. Security & Verification
- **Email/Phone Verification:** Check if the user has verified their contact details via OTP/Link.
- **Password Reset:** As an Admin, you cannot see a user's password, but you can trigger a "Force Password Reset" email if the user is locked out.

### 3. Addresses
- A list of all saved Billing and Shipping addresses. You can edit these on behalf of the user if they request customer support for an address typo.

### 4. Activity & History
The user profile is deeply integrated with the rest of the CMS. From their profile page, you can quickly jump to:
- **Order History:** A list of all purchases they've made.
- **Support Tickets:** Any active or closed customer service requests they've opened.
- **Reviews:** Product or event reviews they've submitted.

*Note on Creation:* Generally, you do not manually create Users in the Admin Panel; users register themselves via your frontend website or mobile app. However, manual creation is available for B2B tele-sales operations.
