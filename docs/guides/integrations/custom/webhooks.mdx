# Webhooks (Real-time Events)

Webhooks allow AVCI CMS to push real-time data to your external systems whenever a specific event occurs, eliminating the need for your ERP to constantly poll the API.

## Configuring Webhooks
1. Go to **Workspace > Settings > Appearance Settings > Webhooks**.
2. Click **Create Webhook**.
3. Provide an **Endpoint URL** (the URL on your server that will receive the POST request).
4. Select the **Events** you want to subscribe to. Common events include:
   - `order.created`: Fires the moment an order is successfully paid.
   - `product.stock_updated`: Fires when inventory changes.
   - `merchant.approved`: Fires when a new seller is onboarded.
5. Provide an optional **Secret Key** to secure your endpoint.

## Handling Webhooks
When an event fires, AVCI CMS sends a POST request with a JSON payload containing the event data. Your server must respond with a `200 OK` status code within 5 seconds.
